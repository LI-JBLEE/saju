## REPORT QUALITY CHECKLIST

Claude API 프롬프트 및 생성 결과물에 대한 품질 기준.

### 콘텐츠 품질
- [ ] 7개 섹션이 모두 포함되어 있는가
- [ ] 각 섹션이 최소 200자 이상인가
- [ ] 성별에 따라 연애/결혼 섹션 내용이 다른가
- [ ] 올해 운세 섹션에 현재 연도가 정확히 반영되어 있는가
- [ ] 일반론이 아닌 해당 사주 고유의 내용이 담겨 있는가
- [ ] 면책 문구가 리포트 하단에 포함되어 있는가

### 사주 계산 정확성
- [ ] 년주 천간·지지 계산이 정확한가 (테스트 케이스 3개 이상)
- [ ] 월주 계산이 절기 기준으로 처리되는가
- [ ] 일주 계산이 60갑자 순환에 따라 정확한가
- [ ] 시주가 null일 때 시주 관련 내용이 리포트에서 제외되는가

### UI/UX 품질
- [ ] 입력 폼에서 잘못된 날짜 입력 시 에러 메시지가 표시되는가
- [ ] 리포트 스트리밍 중 화면이 자연스럽게 업데이트되는가
- [ ] 공유 버튼 클릭 시 "복사됨" 피드백이 표시되는가
- [ ] 모바일(375px)에서 레이아웃 깨짐이 없는가
- [ ] API 에러 시 사용자 친화적 메시지가 표시되는가

## CODE QUALITY STANDARDS

### TypeScript
- `any` 타입 사용 금지 (unknown 또는 명확한 타입 정의)
- 모든 API 응답에 타입 정의 필수
- null / undefined 처리 명시적으로

### Next.js
- Server Component / Client Component 구분 명확히
- 환경 변수 접근은 서버 사이드에서만
- Image 컴포넌트 사용 시 alt 필수

### 일반
- 함수는 단일 책임 원칙 준수
- 100줄 초과 파일은 분리 검토
- 주석은 "왜"를 설명, "무엇"은 코드로
