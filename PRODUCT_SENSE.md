## WHAT IS THIS FILE FOR

에이전트가 기능 구현 중 "이걸 어떻게 해야 하지?"라는 판단이 필요한 순간,
제품 방향성에 맞는 결정을 내릴 수 있도록 돕는 가이드다.
코드 스펙이 아닌 사고방식을 정의한다.

---

## TARGET USER

### 핵심 사용자
사주에 관심 있는 20~40대 한국인.
점집 가기는 부담스럽지만 가볍게 자신의 사주를 알고 싶은 사람.

### 공유 수신자
지인에게 링크를 받아 처음 방문한 사람.
즉시 자신의 사주도 보고 싶어진다.

### 재방문자
예전에 본 리포트를 다시 보거나, 가족/연인의 사주를 대신 조회하는 사람.

## KEY PRODUCT QUESTIONS & ANSWERS

**Q. 리포트 생성이 느리면 어떻게 해야 하나?**
로딩을 숨기지 말고 오히려 활용하라. "사주를 읽는 중"이라는 맥락 안에서 기다림이 기대감이 되도록 LoadingTips로 사주 상식을 보여준다. 스피너만 있는 빈 화면은 절대 허용하지 않는다.

**Q. 사주 계산이 100% 정확하지 않을 수 있는데?**
이 앱은 학술적 정확성보다 사용자 경험을 우선한다. 리포트 하단에 "본 리포트는 AI가 생성한 참고용 콘텐츠입니다"라는 면책 문구를 반드시 포함한다.

**Q. 같은 생년월일시로 두 번 조회하면?**
매번 새로운 리포트를 생성하고 새 URL을 부여한다. 중복 방지 로직을 넣지 않는다. Claude API 응답에 약간의 variability가 있으므로 다른 관점의 리포트가 나올 수 있다는 것이 오히려 장점이다.

**Q. 에러가 발생했을 때 사용자에게 뭘 보여줘야 하나?**
기술적 에러 메시지를 절대 그대로 노출하지 않는다. "사주를 읽는 중 문제가 생겼습니다. 다시 시도해 주세요." 수준의 친근한 문구로 대체한다.

**Q. 기능을 추가해도 될지 판단이 안 설 때는?**
"이게 없어도 핵심 가치(입력→리포트→공유)는 전달되는가?"를 먼저 물어라. YES면 추가하지 않는다. 기능은 줄이는 방향으로 판단한다.

## NORTH STAR METRIC

> "리포트를 받은 사용자가 링크를 공유한다."

모든 기능 결정의 최종 판단 기준은 이 행동을 늘리는가 여부다.
공유를 방해하거나 지연시키는 어떤 요소도 제거 대상이다.

---

## ANTI-PATTERNS (절대 하지 말 것)

- "지금 가입하면 전체 리포트를 볼 수 있어요" 같은 게이팅 — 로그인 없는 원칙 위반
- 광고 배너, 팝업 — 리포트 몰입 방해
- "프리미엄 리포트" 업셀 — 현재 범위 외, 사용자 신뢰 손상
- SNS 공유 버튼 다수 배치 — 링크 복사 하나로 충분
- 리포트 내 외부 링크 — 이탈 유도
