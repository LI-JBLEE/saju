# Application Harness

이 문서는 사주 앱의 application-level harness 범위를 정의합니다.

## 목적

- 제품 문서를 실제 애플리케이션 계약으로 변환한다.
- 구현이 아직 없어도 입력, 리포트, 공유, 장애 대응, 보안 경계를 픽스처와 규칙으로 고정한다.
- 앱 구현이 시작되면 동일한 harness 를 코드/테스트/E2E 검증으로 확장한다.

## 현재 검증 범위

- 생년월일시/성별 입력 검증 규칙
- `saju_data` JSON 구조와 `birth_hour = null` 처리
- 7개 리포트 섹션, 섹션 최소 길이, 올해 운세의 현재 연도 반영
- 성별에 따른 연애/결혼운 섹션 차이
- 면책 문구 존재 여부
- 장애 시나리오별 사용자 메시지와 재시도 정책
- 공유 URL UUID 형식과 영구 보존 정책
- 시크릿 노출 금지와 공개 환경 변수 경계
- 실제 앱 구현 파일/패키지 존재 여부

## 실행 방법

```bash
npm run app:harness
```

전체 검증은 아래 명령으로 실행합니다.

```bash
npm run verify
```

## 현재 기대 결과

- 계약 검증은 통과
- 실제 앱 코드가 아직 없으면 `Implementation Readiness` 는 경고
